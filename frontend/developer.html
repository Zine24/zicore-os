Official SDKs

Start building in minutes with our official client libraries.

Python SDK

Full-featured Python client with async support, type hints, and automatic retries.

pip install zicore-sdk
Async Support Type Hints Auto Retries Streaming
Python
from zicore import ZiCore

client = ZiCore("your_api_key")

# Chat with ZIO AI
response = client.ai.chat(
    message="Calculate Hohmann transfer delta-v",
    context="mission_planning"
)

# Generate 3D model

    type="rocket",
    params={"height": 10, "stages": 2}

# Check wallet balance

REST API

Use any HTTP client. Works with curl, Postman, or any language.

curl -H "Authorization: Bearer $TOKEN" api.zicore.space/v1/...
Universal No Dependencies OpenAPI Spec Webhooks
bash
# Login and get token
TOKEN=$(curl -s -X POST https://api.zicore.space/v1/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"dev@co.com","password":"xxx"}' | jq -r '.token')

# Chat with ZIO
curl -s https://api.zicore.space/v1/api/ai/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"message":"Hello ZIO","context":"general"}'

# Generate 3D model
curl -s https://api.zicore.space/v1/api/3d/generate \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"type":"rocket","params":{"height":10},"format":"glb"}' \
  -o rocket.glb

Handle Authentication

API keys are passed via the Authorization header. Tokens expire after 24 hours — use refresh tokens for long sessions.

python
# Automatic token refresh with the SDK
client = ZiCore(
    api_key="zcs_live_key",
    auto_refresh=True  # SDK handles refresh automatically
)

# Manual token management
tokens = client.auth.login(
    email="dev@company.com",
    password="secure_pass"
)
# tokens.token → access token (24h expiry)
# tokens.refresh → refresh token (30d expiry)

# Refresh when needed
new_tokens = client.auth.refresh(tokens.refresh)

Webhook Events

Get notified in real-time when events happen in your ZiCore apps.

💰

wallet.transaction.completed

Fired when a wallet transfer or payment completes successfully.

🤖

ai.chat.completed

Fired when an AI chat request finishes processing and returns a response.

🧊

materializer.generation.completed

Fired when a 3D model generation request finishes and the file is ready.

🛒

marketplace.purchase.completed

Fired when a marketplace purchase is confirmed and product is delivered.

⚠️

system.alert.triggered

Fired when a system alert or anomaly is detected on your account or services.
